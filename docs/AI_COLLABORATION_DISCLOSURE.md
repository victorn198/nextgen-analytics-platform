# AI Collaboration Disclosure

This repository was built with AI assistance, but not through blind delegation.

## Why I used AI

Modern analytics and product teams increasingly use AI to move faster on:

- implementation
- refactoring
- test drafting
- documentation
- design iteration
- review support

Using AI in a portfolio project is only credible if the work remains explainable
and reviewable. That was the standard I used here.

## What AI helped with

- speeding up repetitive implementation work
- generating alternative UI directions to evaluate
- drafting refactors for the desktop analytics shell
- expanding test coverage around the API and semantic layer
- surfacing security risks and hardening opportunities
- helping draft documentation faster

## What stayed my responsibility

AI did not decide the product direction or business logic. I still owned:

- what the project should demonstrate
- which analyses mattered for BI storytelling
- what to keep or reject
- final review of behavior and design
- whether a feature was defensible in interview terms
- whether claims in the repository remained honest

## Practical framing

The correct way to describe AI usage in this repository is:

> AI was used as an implementation and review accelerator, while product
> decisions, business framing, and final quality control remained manual.

That is a stronger and more honest position than pretending the project was
built without AI or, on the other extreme, handing authorship to AI entirely.

## Why this matters for the portfolio

This repository is intended to show that I can:

- work with modern tools professionally
- use AI pragmatically instead of theatrically
- keep analytics work explainable and testable
- improve speed without dropping accountability
