# GitHub Profile Strategy

This file is private planning material for the GitHub profile around the
portfolio repository.

## Positioning Goal

The profile should communicate:

- analytics engineering fundamentals
- BI and business analysis depth
- Python and SQL delivery
- pragmatic use of AI as an accelerator, not a substitute

## Recommended Pinned Repositories

Pin at most 4 to 6 repositories.

Priority order:

1. `data-pipeline-portfolio`
2. strongest SQL / BI / analytics project after that
3. one automation or Python-heavy repo if it is clean and defensible
4. optionally one smaller but polished repo that shows focus and execution

## What Not To Pin

- unfinished experiments
- repositories with weak documentation
- school-style exercises with little product value
- duplicated ideas with different names
- repos you cannot explain confidently

## Bio / Headline Direction

Keep it concrete. Examples:

### English

- Data Analyst focused on SQL, Python, BI dashboards, and analytics engineering.
- BI and Analytics professional building end-to-end reporting and data products.

### Portuguese

- Analista de Dados com foco em SQL, Python, dashboards BI e analytics engineering.
- Profissional de BI e Analytics com foco em relatorios, automacao e produtos analiticos.

## Profile Hygiene Checklist

- use a clean profile photo
- keep the display name professional
- set location and language accurately
- remove empty or weak profile sections
- make sure pinned repositories have strong README files
- keep contribution graph secondary to repository quality

## Portfolio Story For GitHub Visitors

A visitor should understand this in under one minute:

1. what you build
2. what stack you use
3. what type of role you want
4. which repository proves it best

## Best-Practice Basis

- fewer strong pinned repositories are better than many mediocre ones
- repository clarity matters more than contribution volume
- profile text should match the public work, not overreach beyond it
